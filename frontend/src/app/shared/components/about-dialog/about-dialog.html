What is Common Ground?

A quiz-style way to turn scattered opinions into shared coding standards.

Every team has unwritten opinions on how code should look — and they rarely agree. Common Ground settles them democratically: everyone votes on competing coding styles, the crowd's favorites rise to the top, and you can export the winners as a guide for your AI coding agents. No more style debates in pull requests.

How to use it

  1. 1
    Vote. You're shown a topic — a coding decision like "How should you name boolean variables?" — with a few competing patterns. Pick your favorite, or skip if you have no preference or it's outside your expertise.
  2. 2
    Browse the catalog. Explore every topic and its patterns, filtered by language and other labels. Missing a topic or a better approach? Suggest one — an admin reviews it before it joins the pool.
  3. 3
    See the rankings. Every vote feeds an ELO-style ranking, so the patterns your team genuinely prefers float to the top — backed by data, not by whoever argued loudest.

Why bother?

  • Democratize your standards. Turn the loudest-voice debate into a fair, transparent vote everyone in the organization can take part in.
  • Instruct your AI coding agents. Export the top-ranked patterns as a Markdown rules file so tools like Claude Code write code the way your team actually likes it.
  • Stop re-litigating in reviews. Once the team has spoken, you have a shared reference to point to instead of arguing the same style nits over and over.
